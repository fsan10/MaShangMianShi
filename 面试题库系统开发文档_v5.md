# 面试题库系统开发文档 v5.0

## 一、需求概述

### 1.1 项目背景
将手机上收集的面试问题整理成一个**带后端的完整题库系统**，核心能力：
- **后端服务**：数据持久化、用户管理、RESTful API、防爬虫
- **AI 文档识别**：三个独立接口（面试题 / 笔试题 / 项目），识别文档内容转 JSON，批量入库
- **统计大盘**：排名、知识点、技术栈、题量、分级、考察公司分布、资源（讲解+OJ实操）
- **在线 OJ 实操**：集成开源 SQL 解析器，支持在线编辑和执行
- **参考 CodeBrick**：现代化 UI 设计风格

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| 用户系统 | **微信公众号扫码登录**、权限管理、防爬虫保护 |
| 问题管理 | 面试题 CRUD、笔试题 CRUD、批量导入 |
| AI 识别 - 面试题 | 上传文件 → AI 识别面试问题 → JSON → 批量入库 |
| AI 识别 - 笔试题 | 上传文件 → AI 识别笔试题 → JSON → 批量入库 |
| AI 识别 - 项目 | 上传简历/文档 → AI 提取项目描述、技术栈、项目职责 → JSON → 入库 |
| 项目关联 | 根据简历项目匹配相关面试/笔试问题 |
| 统计大盘 | **排名、知识点、技术栈、题量、分级、考察公司分布、资源** |
| 在线 OJ | **集成开源 SQL 解析器，支持在线编辑和执行** |
| 搜索筛选 | 按分类/难度/来源/项目/题型筛选 |

### 1.3 技术栈选型

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 前端 | React 18 + TypeScript + Ant Design | 现代化 UI 框架 |
| 后端 | Python FastAPI | 高性能异步框架 |
| 数据库 | PostgreSQL + Redis | 主库 + 缓存 |
| AI 模型 | **用户自定义配置**，不绑定默认模型 | 支持任意 OpenAI 兼容 API |
| 在线 SQL | **sql.js (SQLite)** + **sql-online-ide** | 浏览器端执行，无需后端 |
| 防爬虫 | Redis 限流 + 验证码 + 请求签名 | 多层防护 |
| 部署 | Docker + Nginx | 容器化部署 |

### 4.5 数据表设计

```sql
-- 知识点表 (knowledge_points)
CREATE TABLE knowledge_points (
    id SERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,           -- 知识点名称
    category_id INTEGER REFERENCES categories(id),
    tech_stack TEXT[],                    -- 技术栈标签 ["MySQL", "Hive"]
    difficulty_level VARCHAR(20),         -- 分级：初阶、中阶、高阶
    question_count INTEGER DEFAULT 0,     -- 关联题量
    sort_order INTEGER DEFAULT 0,         -- 排序
    has_explanation BOOLEAN DEFAULT false, -- 是否有讲解
    has_oj_practice BOOLEAN DEFAULT false, -- 是否有OJ实操
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 知识点-问题关联表
CREATE TABLE knowledge_point_questions (
    id SERIAL PRIMARY KEY,
    knowledge_point_id INTEGER REFERENCES knowledge_points(id),
    question_id INTEGER REFERENCES questions(id),
    UNIQUE(knowledge_point_id, question_id)
);

-- 公司-知识点关联统计（考察公司分布）
CREATE TABLE company_knowledge_stats (
    id SERIAL PRIMARY KEY,
    company_name VARCHAR(100) NOT NULL,
    knowledge_point_id INTEGER REFERENCES knowledge_points(id),
    question_count INTEGER DEFAULT 0,
    frequency_score INTEGER DEFAULT 0,    -- 考察频率得分（用于方块颜色深浅）
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(company_name, knowledge_point_id)
);
```

## 五、在线 OJ 实操设计

### 5.1 技术选型

经过调研，推荐以下开源方案：

| 方案 | 项目 | 特点 | 适用场景 |
|------|------|------|---------|
| **推荐** | [sql.js](https://github.com/sql-js/sql.js) | SQLite 编译为 WebAssembly，纯浏览器端执行 | 轻量级 SQL 练习，无需服务器 |
| **推荐** | [sqlite-online](https://github.com/vwh/sqlite-online) | 基于 sql.js 的完整在线 IDE | 可直接嵌入或参考实现 |
| 备选 | [sql-online-ide](https://github.com/jeremy6680/sql-online-ide) | 支持多数据库（SQLite、MySQL、PostgreSQL） | 需要后端支持多数据库 |
| 备选 | [sqlpad](https://github.com/sqlpad/sqlpad) | 功能强大的 SQL 编辑器 | 独立部署，功能较重 |

### 5.2 推荐方案：sql.js + 自定义 OJ 组件

**核心优势**：
- ✅ 纯浏览器端执行，无需后端数据库连接
- ✅ 零服务器成本，无安全风险
- ✅ 启动快，用户体验好
- ✅ 支持导入自定义数据库文件
- ✅ MIT 开源协议

**实现方式**：

### 5.4 前端 OJ 组件实现

```typescript
// components/SqlPlayground/index.tsx
import { useEffect, useRef, useState } from 'react';
import initSqlJs from 'sql.js';

interface SqlPlaygroundProps {
  initSql: string;           // 初始化 SQL
  expectedColumns?: string[]; // 期望的列名
  onSuccess?: () => void;
  onError?: (error: string) => void;
}

export const SqlPlayground: React.FC<SqlPlaygroundProps> = ({
  initSql,
  expectedColumns,
  onSuccess,
  onError
}) => {
  const [db, setDb] = useState<any>(null);
  const [sqlCode, setSqlCode] = useState('');
  const [result, setResult] = useState<any>(null);
  const [error, setError] = useState('');

  useEffect(() => {
    // 初始化 sql.js
    initSqlJs({
      locateFile: file => `https://sql.js.org/dist/${file}`
    }).then(SQL => {
      const database = new SQL.Database();
      // 执行初始化 SQL
      database.run(initSql);
      setDb(database);
    });
  }, [initSql]);

  const handleExecute = () => {
    if (!db) return;
    
    try {
      const res = db.exec(sqlCode);
      setResult(res);
      setError('');
      
      // 验证结果
      if (expectedColumns && res.length > 0) {
        const columns = res[0].columns;
        const isMatch = expectedColumns.every(col => columns.includes(col));
        if (isMatch) {
          onSuccess?.();
        } else {
          onError?.('结果列不匹配');
        }
      }
    } catch (err: any) {
      setError(err.message);
      setResult(null);
      onError?.(err.message);
    }
  };

  return (
    <div className="sql-playground">
      <div className="editor">
        <textarea
          value={sqlCode}
          onChange={e => setSqlCode(e.target.value)}
          placeholder="输入 SQL 查询..."
        />
      </div>
      <button onClick={handleExecute}>运行</button>
      {error && <div className="error">{error}</div>}
      {result && <ResultTable data={result} />}
    </div>
  );
};
```

```typescript
  useEffect(() => {
    if (!session || status === 'success' || status === 'expired') return;
    
    const timer = setInterval(async () => {
      const result = await checkLoginStatus(session.session_id);
      setStatus(result.status);
      
      if (result.status === 'success') {
        localStorage.setItem('token', result.token);
        // 跳转到首页
        navigate('/');
      }
    }, 2000);
    
    return () => clearInterval(timer);
  }, [session, status]);

  // 3. 倒计时
  useEffect(() => {
    if (countdown <= 0) {
      setStatus('expired');
      return;
    }
    const timer = setInterval(() => {
      setCountdown(c => c - 1);
    }, 1000);
    return () => clearInterval(timer);
  }, [countdown]);

  // 4. 复制验证码
  const handleCopy = () => {
    navigator.clipboard.writeText(session.verify_code);
    message.success('验证码已复制');
  };

  // 5. 刷新二维码
  const handleRefresh = () => {
    setStatus('pending');
    fetchLoginQRCode().then(setSession);
  };

  return (
    <Modal visible={true}>
      <QRCode value={session?.qrcode_url} />
      <div className="verify-code">
        <span>{session?.verify_code}</span>
        <Button onClick={handleCopy}>复制</Button>
      </div>
      <Steps />
      <StatusIndicator status={status} countdown={countdown} />
      {status === 'expired' && <Button onClick={handleRefresh}>刷新</Button>}
    </Modal>
  );
};
```

### 11.2 后端服务实现

```python
# services/wechat_service.py

import hashlib
import xml.etree.ElementTree as ET
from datetime import datetime, timedelta
import random
import string

class WechatLoginService:
    """微信公众号登录服务"""

    def __init__(self, db: AsyncSession):
        self.db = db

    async def generate_login_code(self) -> dict:
        """生成登录二维码和验证码"""
        # 生成 6 位数字验证码
        code = ''.join([str(random.randint(0, 9)) for _ in range(6)])
        
        # 生成会话 ID
        session_id = hashlib.md5(f"{code}{time.time()}".encode()).hexdigest()
        
        # 生成带参数二维码 URL
        qrcode_url = await self._create_qrcode(session_id)
        
        # 存入数据库
        login_code = LoginCode(
            code=code,
            session_id=session_id,
            status='pending',
            expires_at=datetime.now() + timedelta(minutes=5)
        )
        self.db.add(login_code)
        await self.db.commit()
        
        return {
            'session_id': session_id,
            'qrcode_url': qrcode_url,
            'verify_code': code,
            'expires_in': 300
        }

    async def handle_wechat_message(self, xml_data: str) -> str:
        """处理微信消息回调"""
        # 解析 XML
        root = ET.fromstring(xml_data)
        openid = root.find('FromUserName').text
        content = root.find('Content').text.strip()
        
        # 判断是否为 6 位数字验证码
        if len(content) == 6 and content.isdigit():
            # 查询验证码
            result = await self.db.execute(
                select(LoginCode).where(
                    LoginCode.code == content,
                    LoginCode.status == 'pending',
                    LoginCode.expires_at > datetime.now()
                )
            )
            login_code = result.scalar_one_or_none()
            
            if login_code:
                # 更新验证码状态
                login_code.status = 'used'
                login_code.openid = openid
                
                # 创建或更新用户
                user = await self._get_or_create_user(openid)
                
                await self.db.commit()
                
                return self._make_text_response(openid, "登录成功！欢迎来到面试题库~")
            else:
                return self._make_text_response(openid, "验证码无效或已过期，请重新获取")
        
        # 其他消息，返回帮助提示
        return self._make_text_response(
            openid, 
            "欢迎关注面试题库！\n请发送登录页面显示的 6 位验证码完成登录。"
        )

    async def check_login_status(self, session_id: str) -> dict:
        """检查登录状态"""
        result = await self.db.execute(
            select(LoginCode).where(LoginCode.session_id == session_id)
        )
        login_code = result.scalar_one_or_none()
        
        if not login_code:
            return {'status': 'invalid', 'message': '无效的会话'}
        
        if login_code.status == 'used' and login_code.openid:
            # 登录成功，生成 JWT
            user = await self._get_user_by_openid(login_code.openid)
            token = create_jwt_token(user.id)
            
            return {
                'status': 'success',
                'token': token,
                'user': {
                    'id': user.id,
                    'nickname': user.nickname,
                    'avatar_url': user.avatar_url
                }
            }
        
        if login_code.expires_at < datetime.now():
            return {'status': 'expired', 'message': '验证码已过期'}
        
        return {'status': 'pending', 'message': '等待扫码...'}
```
