# 面试题库系统开发文档 v6.0

## 一、需求概述

### 1.1 项目背景
将手机上收集的面试问题整理成一个**带后端的完整题库系统**，核心能力：
- **后端服务**：数据持久化、用户管理、RESTful API、防爬虫
- **AI 文档识别**：三个独立接口（面试题 / 笔试题 / 项目），识别文档内容转 JSON，批量入库
- **统计大盘**：排名、知识点、技术栈、题量、分级、考察公司分布、资源（讲解+OJ实操）
- **在线 OJ 实操**：集成开源 SQL 解析器，支持在线编辑和执行
- **学习进度**：个人学习记录、365天打卡日历、章节进度追踪
- **巩固功能**：按知识点逐个攻克、智能推荐薄弱点
- **错题本**：活跃错题管理、已出本记录、错题重做
- **参考 CodeBrick**：现代化 UI 设计风格

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| 用户系统 | **微信公众号扫码登录**、权限管理、防爬虫保护 |
| 问题管理 | 面试题 CRUD、笔试题 CRUD、批量导入 |
| AI 识别 - 面试题 | 上传文件 → AI 识别面试问题 → JSON → 批量入库 |
| AI 识别 - 笔试题 | 上传文件 → AI 识别笔试题 → JSON → 批量入库 |
| AI 识别 - 项目 | 上传简历/文档 → AI 提取项目描述、技术栈、项目职责 → JSON → 入库 |
| 项目关联 | 根据简历项目匹配相关面试/笔试问题 |
| 统计大盘 | **排名、知识点、技术栈、题量、分级、考察公司分布、资源** |
| 在线 OJ | **集成开源 SQL 解析器，支持在线编辑和执行** |
| **学习进度** ⭐ | **个人作战记录、365天打卡日历、章节进度追踪** |
| **巩固功能** ⭐ | **按知识点逐个攻克、智能推荐薄弱点** |
| **错题本** ⭐ | **活跃错题管理、已出本记录、错题重做** |
| 搜索筛选 | 按分类/难度/来源/项目/题型筛选 |

### 1.3 技术栈选型

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 前端 | React 18 + TypeScript + Ant Design | 现代化 UI 框架 |
| 后端 | Python FastAPI | 高性能异步框架 |
| 数据库 | PostgreSQL + Redis | 主库 + 缓存 |
| AI 模型 | **用户自定义配置**，不绑定默认模型 | 支持任意 OpenAI 兼容 API |
| 在线 SQL | **sql.js (SQLite)** + **sql-online-ide** | 浏览器端执行，无需后端 |
| 防爬虫 | Redis 限流 + 验证码 + 请求签名 | 多层防护 |
| 部署 | Docker + Nginx | 容器化部署 |
